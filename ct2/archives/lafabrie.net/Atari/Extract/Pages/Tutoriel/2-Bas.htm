Travailler avec Icon Extract
 
Icon Extract permet de traiter toutes les icônes du monde Windows et du monde Linux. Tous les types sont supportés, de 1 à 32 bits. Les icônes supérieures à 8 bits seront converties en 256 couleurs. L'AES de l'ATARI ne permet (pas encore) d'utiliser des icônes au dela de 256 couleurs. Traiter des icônes en 24 ou 32 bits nécessite du temps de calcul important. Il existe souvent dans les fichiers ICO et CUR des icônes prévues pour chaque résolution, il n'est donc pas nécessaire de tout traiter. Vous disposez d'une aide sous Bubble Gem (si celui-ci est installé) dans plusieurs fenêtres du programme, elle est activable avec le bouton droit de la souris.

Voici les types d'icônes supportés :
ICO Icône Windows (Icon File).
CUR Curseur Windows (Cursor File).
ANI Curseur animé Windows (Animated File), contenant des icônes ICO ou CUR.
ICL Librairie d'icônes au format ICO ou CUR.
DLL Librairie dynamique de Windows (Ressource File), pouvant contenir des icônes ICO ou CUR.
EXE Executable Windows (non encore disponible), pouvant contenir des icônes ICO ou CUR.
OCX Ressource Windows (non encore disponible), pouvant contenir des icônes ICO ou CUR.
XPM Icône Linux.
 
L'interface de travail

Elle est très intuitive. Après avoir cliqué sur l'onglet "CONFIG" :

Sélectionner le type de fichier à traiter ainsi que les tailles et les types à conserver.

Cocher la case "Palette Icône" pour relire celle-ci dans le cas ou chaque icône ne dispose pas de la même.

Cocher une des cases "Palette 256 couleurs", pour le transfert des palettes. Sélectionner une méthode de calcul : normal (1) ouprécise (2).

Indiquer dans "True Color" l'icône à conserver dans le cas ou celle-ci existe en (24-32 bits) et en 8 bits,

En option, vous pouvez renommer les fichiers de plus de 12 caractères (Windows ou Linux) pour les rendre compatible avec le TOS. Ils seront renommés de 00001.xxx à 99999. En cas d'erreur de traitement, vous pouvez créer un fichier de debugage. Pour terminer, vous pouvez fixer le nombre d'icônes pour le fichier RSC.

Tramage sous nvdi 5

Dans le cas où NVDI 5.xx est présent sur votre ordinateur, vous pouvez choisir d'utiliser ses routines de tramage et de transfert MFDB. Par exemple : si une icône est présente seulement en 256 couleurs, NVDI calculera alors son équivalent en 16 et 2 couleurs. Pour obtenir le meilleur résultat, mais plus long en calcul, vous devez sélectionner les options suivantes:

Type de tramage : T LOGIC MODE
Qualité du résultat : PRECIS

Pour le moment, les tailles d'icônes de 72x72 à 128x128 pixels ne sont plus traitées par les routines NVDI. En effet les allocations mémoires font "crascher" le système sous MAGIC dans les modes graphiques supérieurs à 8 bits.
